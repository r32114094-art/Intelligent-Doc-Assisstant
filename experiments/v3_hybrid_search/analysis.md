## 核心发现

### ✅ 发现 1: 评估修复带来巨幅提升

CR 从 V1 的 2.05 飙升至 V3 的 4.05（+98%），**主因是评估 Bug 修复而非架构变更**：

- **Bug 1 (截断)**: Judge 之前只看 chunk 前 300 字符（约 1/4），严重低估了检索质量
- **Bug 2 (二次检索)**: Judge 拿另一次检索的 context 给 answer 打分，FF 被严重低估
- **Bug 3 (双重调用)**: MQE/HyDE 调了两次，浪费 API 但结果不一致

修复后 Baseline 即达 4.47 综合分，说明 V1 的检索能力本就不差，只是被错误的评估方法掩盖了。

### ✅ 发现 2: FF（忠实度）全面达标

| 配置 | V1 FF | V3 FF |
|---|---|---|
| Baseline | 3.55 | **4.85** |
| +MQE+HyDE | 3.85 | **5.00** |

- 修复 Bug 2 后，Judge 看到的 context 与 LLM 生成 answer 用的 context 完全一致
- +MQE+HyDE 配置下 FF 达到完美 5.00，说明 LLM 严格基于检索内容回答

### 🏆 发现 3: 最佳配置为 +MQE+HyDE

- **综合 4.57/5.00**，CR=4.10，FF=5.00，AR=4.60
- MQE 扩展了查询覆盖面，HyDE 缩短了 query-document 语义鸿沟
- 两者组合在中等难度题上表现最佳（CR=4.50，AR=4.90）

### ⚠️ 发现 4: Rerank 仍然有害

| 指标 | 无 Rerank | +Rerank | 差值 |
|---|---|---|---|
| CR | 4.05 | **2.95** | **-27%** |
| AR | 4.50 | **3.70** | **-18%** |
| FF | 4.85 | 4.95 | +2% |

Rerank 的 Cross-Encoder（`ms-marco-MiniLM-L-6-v2`）是英文模型，在中文语料上：
- 过度过滤了实际相关的 chunk（CR 大幅下降）
- 导致 LLM 缺少关键上下文，AR 随之下降
- 唯一好处是略微提升 FF（过滤掉噪声 chunk）

**结论**: 当前 Reranker 不适合中文场景，建议替换为中文 Cross-Encoder 或直接禁用。

### 📊 发现 5: 困难题 CR 仍有提升空间

| 难度 | CR |
|---|---|
| easy | 4.20 |
| medium | **4.50** |
| hard | 3.20 |

困难题 CR 仅 3.20，原因：
- 多数困难题需要跨 chunk 综合推理（如 Q16 跨 4 个模型比较 Pre-Norm/Post-Norm）
- 当前 chunk_size=800 可能导致相关信息分散在多个 chunk 中
- BM25 对专业术语的覆盖能力有限

## 总结与下一步

### 已解决的问题
- ✅ CR 虚低 → 评估修复后真实 CR 达 4.05+
- ✅ FF 虚低 → 统一 context 来源后 FF 达 4.85+
- ✅ MQE/HyDE 重复调用 → 统一由 Retriever 处理
- ✅ ask_streaming 与 ask 路径不一致 → 统一走 Retriever

### 待解决的问题（V4 方向）
- **P0**: 替换中文 Reranker（如 `BAAI/bge-reranker-base`）或禁用
- **P1**: 困难题 CR 优化 — 考虑减小 chunk_size 或引入 chunk 重叠增强
- **P1**: 引入 BM25 权重调优（当前 Dense:BM25 = 1:1 via RRF）
- **P2**: 多 Judge 交叉验证 — 减少单模型评分偏见
